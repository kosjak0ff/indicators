# T-003: Add manual daily indicator overrides

## Goal

- Allow the daily digest to use manually supplied indicator readings for a specific run.
- Keep the current automatic fetch flow intact for indicators that are not overridden.

## Scope

- Add a CLI option for loading manual indicator readings from a JSON file.
- Support partial overrides so a single source such as CoinGlass can be supplied manually.
- Cover the override loading and digest generation path with tests.

## Verification

- `.venv/bin/python -m pytest -q`

## Implemented

- Added manual indicator override loading in [indicators_digest/overrides.py](/home/code/codex-projects/indicators/indicators_digest/overrides.py) for `altcoin_season`, `fear_greed`, and `bull_market_peak`.
- Extended [indicators_digest/cli.py](/home/code/codex-projects/indicators/indicators_digest/cli.py) with `run --manual-indicators <path>` so a JSON file can replace live fetches for a single run.
- Updated [indicators_digest/app.py](/home/code/codex-projects/indicators/indicators_digest/app.py) to support partial overrides, keeping live requests for any indicator not present in the JSON payload.
- Added regression coverage in [tests/test_overrides.py](/home/code/codex-projects/indicators/tests/test_overrides.py) and [tests/test_app.py](/home/code/codex-projects/indicators/tests/test_app.py) for override parsing and the manual CoinGlass fallback path.
- Updated [README.md](/home/code/codex-projects/indicators/README.md) with an example manual override JSON payload and the corresponding CLI command.

## Notes

- The override file is intentionally partial: supplying only `bull_market_peak` skips the CoinGlass fetch while the other indicators still come from live sources.
- Verification passed with `.venv/bin/python -m pytest -q` on 2026-03-31.
